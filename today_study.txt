📚 오늘의 단어 (2026-04-06)

1. consequence / 결과, 영향 / 컨시퀀스
2. investigate / 조사하다 / 인베스티게이트
3. dependency / 의존성 / 디펜던시
4. recovery / 회복 / 리커버리
5. generous / 관대한, 넉넉한 / 제너러스
6. significant / 중요한, 의미 있는 / 시그니피컨트
7. vulnerable / 취약한 / 벌너러블
8. prioritize / 우선순위를 정하다 / 프라이오리타이즈
9. constraint / 제약 조건 / 컨스트레인트
10. enthusiasm / 열정 / 인쑤지애즘

────────────────────────

1. consequence (결과, 영향) / 컨시퀀스
* Ignoring the warning message can have serious consequences for the system.
  → 경고 메시지를 무시하면 시스템에 심각한 결과를 초래할 수 있다.

────────────────────────

2. investigate (조사하다) / 인베스티게이트
* The team needs to investigate why the deployment failed last night.
  → 팀은 어젯밤 배포가 실패한 이유를 조사해야 한다.

────────────────────────

3. dependency (의존성) / 디펜던시
* The project has too many external dependencies that slow down the build.
  → 프로젝트에 빌드를 느리게 하는 외부 의존성이 너무 많다.

────────────────────────

4. recovery (회복) / 리커버리
* The disaster recovery plan helped restore the service within an hour.
  → 재해 복구 계획 덕분에 한 시간 내에 서비스를 복구했다.

────────────────────────

5. generous (관대한, 넉넉한) / 제너러스
* The company offers a generous vacation policy to all employees.
  → 회사는 모든 직원에게 넉넉한 휴가 정책을 제공한다.

────────────────────────

6. significant (중요한, 의미 있는) / 시그니피컨트
* There was a significant improvement in response time after the update.
  → 업데이트 후 응답 시간이 크게 개선되었다.

────────────────────────

7. vulnerable (취약한) / 벌너러블
* The old version of the library is vulnerable to security attacks.
  → 라이브러리의 이전 버전은 보안 공격에 취약하다.

────────────────────────

8. prioritize (우선순위를 정하다) / 프라이오리타이즈
* You should prioritize critical bugs over new feature requests.
  → 새 기능 요청보다 치명적인 버그를 우선순위에 두어야 한다.

────────────────────────

9. constraint (제약 조건) / 컨스트레인트
* Budget constraints forced us to choose a simpler solution.
  → 예산 제약으로 인해 더 간단한 솔루션을 선택할 수밖에 없었다.

────────────────────────

10. enthusiasm (열정) / 인쑤지애즘
* Her enthusiasm for learning new languages impressed everyone.
  → 새로운 언어를 배우려는 그녀의 열정은 모두에게 깊은 인상을 주었다.
